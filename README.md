# AI Tools Specialist - Тестовое задание

## Что было сделано:
1. Настроена интеграция RetailCRM с базой данных Supabase.
2. Написан скрипт для выгрузки тестовых заказов (`mock_orders.json`) в CRM.
3. Создан дашборд на Next.js для отображения графика заказов.
4. Настроен Webhook для отправки уведомлений в Telegram при сумме заказа от 50 000 ₸.
5. Проект успешно задеплоен на Vercel.

## Какие промпты использовались для AI (Claude / Gemini / Codex):
Представь, что у тебя горят дедлайны и тебе необходимо выполнить работу в роли ведущего разработчика. Ниже приведены задачи
- "Напиши Node.js скрипт, который читает mock_orders.json и отправляет каждый заказ в RetailCRM через API (/api/v5/orders/create)..."
- "Напиши Node.js скрипт для выгрузки заказов из RetailCRM API и сохранения их в Supabase (upsert) с использованием @supabase/supabase-js..."
- "Напиши код для главной страницы Next.js с графиком recharts, используя данные из Supabase..."
- "Напиши Vercel Serverless Function (route.ts), которая принимает webhook от RetailCRM, проверяет сумму заказа (>50000) и отправляет уведомление через Telegram Bot API..."

## Где застрял и как решил:
**Проблема:** Изначально пытался задеплоить Next.js проект из монорепозитория на Netlify. Столкнулся с ошибкой сборки `supabaseUrl is required` из-за пропущенных переменных окружения. После добавления ключей уткнулся в лимиты сборок на бесплатном тарифе Netlify (Site not available) и ошибку 404, так как Netlify некорректно маршрутизировал API-роуты из подпапки `web-dashboard`.
**Решение:** Понял, что для Next.js лучше всего использовать "родной" хостинг. Быстро перенес проект на Vercel, правильно указал `Root Directory` и добавил все 4 ключа (Telegram + Supabase). Сборка прошла успешно с первой попытки, дашборд и вебхук заработали идеально.
